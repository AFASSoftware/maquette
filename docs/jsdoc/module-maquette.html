Module: maquette

The main object in maquette is the maquette object. It is either bound to window.maquette or it can be obtained using browserify or requirejs.

(static) h(selector, propertiesopt, childrenopt) → {VNode}

The h method is used to create a virtual DOM node. This function is largely inspired by the mercuryjs and mithril frameworks. The h stands for (virtual) hyperscript.

Parameters:
Name Type Attributes Description
selector string

Contains the tagName, id and fixed css classnames in CSS selector format. It is formatted as follows: tagname.cssclass1.cssclass2#id.
properties Object <optional>

An object literal containing properties that will be placed on the DOM node.

Properties
Name Type Description
* function

Properties with functions values like onclick:handleClick are registered as event handlers
* String

Properties with string values, like href:"/" are used as attributes
* object

All non-string values are put on the DOM node as properties
key Object

Used to uniquely identify a DOM node among siblings. A key is required when there are more children with the same selector and these children are added or removed dynamically.
classes Object

An object literal like {important:true} which allows css classes, like important to be added and removed dynamically.
styles Object

An object literal like {height:"100px"} which allows styles to be changed dynamically. All values must be strings.
enterAnimation string | enterAnimationCallback

The animation to perform when this node is added to an already existing parent. More about animations. When this value is a string, you must pass a projectionOptions.transitions object when creating the projector module:maquette.createProjector.
exitAnimation string | exitAnimationCallback

The animation to perform when this node is removed while its parent remains. When this value is a string, you must pass a projectionOptions.transitions object when creating the projector module:maquette.createProjector. More about animations.
updateAnimation updateAnimationCallback

The animation to perform when the properties of this node change. This also includes attributes, styles, css classes. This callback is also invoked when node contains only text and that text changes. More about animations.
afterCreate afterCreateCallback

Callback that is executed after this node is added to the DOM. Childnodes and properties have already been applied.
afterCreate afterUpdateCallback

Callback that is executed every time this node may have been updated. Childnodes and properties have already been updated.
children Array.<Object> <optional>

An array of virtual DOM nodes to add as child nodes. This array may contain nested arrays, null or undefined values. Nested arrays are flattened, null and undefined will be skipped.
Returns:

A VNode object, used to render a real DOM later. NOTE: There are three basic rules you should be aware of when updating the virtual DOM.
